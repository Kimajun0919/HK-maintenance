## 한 장 요약

### 지금 상태

- 코퍼스: 문서 135개 / 400KB / 청크 793개 / 고객사 61개 — **매우 작습니다**
- 하이브리드 검색 top-3 정확도 **0.633**
- 인증·권한·감사 **전무** (모든 API 무인증)

### 가장 중요한 발견 3가지

**1. 임베딩이 실제 모델이 아닙니다.**
배포된 모든 환경이 `EMBEDDING_BACKEND=hash` — blake2b 해시 랜덤 프로젝션입니다. 지금까지 "벡터 검색"이라 불린 것은 의미 기반 검색이 아니었습니다. 평가 리포트의 `vector_only 0.467`은 임베딩 모델 성능이 아니라 랜덤 프로젝션 성능입니다.

**2. Recall 천장 — 랭킹이 아니라 회수 문제입니다.**
모든 검색 모드에서 top-3와 top-5가 **완전히 동일**합니다. 4~5위에서 추가로 맞은 케이스가 0건. 순위를 개선해도 소용없고, 정답이 후보 집합에 애초에 들어오지 않습니다.

**3. Graph RAG는 만능이 아니며, 어떤 그래프인지가 결정적입니다.**
통제 실험(arXiv 2502.11371)에서 LLM이 만든 지식그래프는 일반 RAG보다 검색 정확도가 낮았고(0.392 vs 0.886), 정답 엔티티의 **65.8%만 포착**했으며, 정답 없는 질문에서 **80% 이상 환각**했습니다.

반면 HK-maintenance에는 이미 100% 정확한 관계 데이터(고객사 폴더, 접수 테이블의 담당자·유형·상태)가 있으므로, **LLM에게 그래프를 만들게 할 이유가 없습니다.** 규칙 기반 추출의 현재 커버리지는 **42.5%**이며(초기 PoC의 89.8%는 범용어 하나가 만든 과대평가였고 정정했습니다), 사전 확충으로 올릴 수 있는 반면 LLM 추출은 누락 지점을 특정할 수조차 없습니다.

### 권고 순서

**우선순위(P)**는 "무엇이 중요한가", **단계(Phase)**는 "언제 하는가"입니다. 두 축을 혼동하지 않도록 대응 관계를 명시합니다.

| 우선순위 | 내용 | 실행 단계 |
|---|---|---|
| **P0-α** | 코퍼스 오염 제거 (메타 문서 5개) — 반나절 | Phase 0 |
| **P0** | 평가셋 150건 구축 + 실패 케이스 원인 분류 | Phase 0 |
| **P1** | 검색 정상화 (KURE-v1 + Kiwi + FTS + 리랭커) — **여기까지가 진짜 베이스라인** | Phase 1 |
| **P2** | G1 결정론적 그래프 | Phase 4 |
| **P3** | 보안 계층 (인증·RBAC·RLS·감사) | Phase 2–3 (병렬) |
| **P4** | 코드베이스 정리 (저메모리 우회 로직 삭제, 무음 실패 제거 등) | Phase 1·6 분산 |
| **P5** | G2 LLM 보강 그래프 — **측정 결과 좋을 때만** | Phase 5 (조건부) |

Phase 번호가 P 번호보다 먼저 오는 구간이 있는 이유는, **보안과 온프레미스 이전이 그래프보다 리드타임이 길기 때문**입니다(하드웨어 조달, 망분리 정책 협의). 중요도는 그래프가 높지만 착수는 먼저 해야 합니다.

**P1 없이 그래프부터 가면 개선 효과의 귀속이 불가능합니다.** 임베딩만 고쳐도 상당한 개선이 예상되는데, 이를 그래프 도입 성과로 오인하면 이후 의사결정이 왜곡됩니다.

### 핵심 기술 선택

| 계층 | 선정 | 이유 |
|---|---|---|
| 전체 스택 | **PostgreSQL 단일** | 이 규모에서 전용 벡터/그래프 DB는 과잉. 망분리에선 백업·패치 대상 하나가 최고 가치 |
| 임베딩 | **KURE-v1** (MIT) | MTEB-ko-retrieval 1위. OpenAI text-embedding-3-large보다 한국어 성능 우위 |
| LLM | **Qwen3.5-27B** (Apache 2.0) | ⚠️ **EXAONE은 비상업 라이선스** — LG 자체 리포트에서도 Qwen3.5-27B가 한국어 전 벤치 상회 |
| 서빙 | **vLLM** | Ollama 대비 동시 8사용자 기준 2.2~4배 처리량. TGI는 2025-12 유지보수 모드 |
| GPU | **48GB 1장** (L40S / A6000 Ada) | LLM+임베딩+리랭커를 한 장에. ⚠️ GeForce는 데이터센터 EULA 위반 소지 |

### 즉시 확인이 필요한 미결 사항

- [ ] **동시 사용자 수** — 인프라 산정의 최대 미확정 변수
- [ ] **모델 라이선스 법무 검토** — EXAONE 비상업 이슈, 리드타임 김
- [ ] **실패 11건이 콘텐츠 부재 때문인가** — 그렇다면 Graph RAG로 해결 불가
- [ ] **L4(관계형) 질의 수요가 실제로 있는가** — 유지보수팀 인터뷰 필요. 없으면 그래프 투자 근거 약화
- [ ] 망분리 패치 반입 절차 — 구축은 되나 6개월 뒤 패치 못 하는 사태 방지
